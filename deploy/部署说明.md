```bash

apt update

# 安装 Docker Compose（独立版本）

    -o /usr/local/bin/docker-compose
chmod +x /usr/local/bin/docker-compose

# 验证
docker --version
docker compose version
```

## 三、方式一：服务器上直接构建（推荐，有网络环境）

### 3.1 上传项目

```bash
# 在服务器上
mkdir -p /opt/oa-agent
cd /opt/oa-agent

# 上传整个项目（排除 .git、node_modules、data 等）
# 方式 A：git clone
git clone <仓库地址> .
# 方式 B：scp 上传
# scp -r ./* user@server:/opt/oa-agent/
```

### 3.2 配置环境变量

```bash
cd /opt/oa-agent/deploy
cp .env.example .env
vim .env   # 填入 LLM_API_KEY 等必要配置
```

**`.env` 必填项：**

```ini
LLM_API_KEY=sk-xxxxxxxx          # 替换为真实 API Key
LLM_BASE_URL=https://api.deepseek.com/v1
LLM_MODEL_NAME=deepseek-chat
# OA 对接（后续填写）
SEEYON_OA_BASE_URL=
SEEYON_OA_API_TOKEN=
```

### 3.3 构建并启动

```bash
cd /opt/oa-agent/deploy

# 构建镜像（首次约 5～15 分钟，取决于网络）
docker compose build

# 后台启动
docker compose up -d

# 查看运行状态
docker compose ps

# 查看日志
docker compose logs -f --tail=50
```

### 3.4 验证部署

```bash
# 健康检查
curl http://localhost/health

# 预期返回：
# {"status":"ok","version":"1.0.0","llm_model":"deepseek-chat",...}

# 浏览器访问
# http://服务器IP
```

## 四、方式二：本地构建 & 离线部署（客户服务器无网络）

### 4.1 本地构建（Windows / macOS）

```powershell
# 在项目根目录
.\deploy\build.ps1 -ExportImages -Tag 1.0.0
```

执行后生成 `oa-agent-images-1.0.0.tar`。

### 4.2 打包部署文件

手动打包以下文件：

```
oa-agent-deploy/
├── oa-agent-images-1.0.0.tar    # Docker 镜像
├── docker-compose.yml            # 容器编排
├── .env                          # 生产配置（已填写）
├── nginx.conf                    # Nginx 配置（备用）
└── 部署说明.md                    # 本文档
```

### 4.3 传输到服务器

```bash
# 打包
tar czf oa-agent-deploy.tar.gz oa-agent-deploy/

# 上传
scp oa-agent-deploy.tar.gz user@server:/opt/
```

### 4.4 服务器上导入启动

```bash
cd /opt
tar xzf oa-agent-deploy.tar.gz
cd oa-agent-deploy

# 导入镜像
docker load -i oa-agent-images-1.0.0.tar

# 确认
docker images | grep oa-agent

# 启动
docker compose up -d

# 验证
curl http://localhost/health
```

## 七、更新部署

```bash
cd /opt/oa-agent/deploy

# 拉取最新代码 / 上传新文件

# 无缓存重新构建
docker compose build --no-cache

# 停旧启新
docker compose down
docker compose up -d

# 清理旧镜像（可选）
docker image prune -a
```

### 仅修改 docker-compose.yml（不改代码/Dockerfile）

修改端口、环境变量、卷挂载等配置不需要重新构建镜像：

```bash
docker compose down
# 编辑 docker-compose.yml
docker compose up -d
```

## 八、Docker 缓存与磁盘清理

### 8.1 构建缓存

```bash
# 查看构建缓存占用
docker builder du

# 清理构建缓存（保留最近 10 天）
docker builder prune

# 清理全部构建缓存
docker builder prune -a
```

### 8.2 清理无用资源

```bash
# 查看 Docker 磁盘占用
docker system df

# 清理悬空镜像（<none>:<none>）
docker image prune

# 清理所有未使用的镜像（含旧版本）
docker image prune -a

# 清理停止的容器、未使用的网络、悬空镜像
docker system prune

# 彻底清理（含所有未使用的镜像 + 构建缓存）
docker system prune -a --volumes
```

> ⚠️ `--volumes` 会删除未被任何容器使用的数据卷，确认后再执行。

### 8.3 中断构建 & 重来

```bash
# 直接 Ctrl+C 中断，然后：
docker compose down          # 停止残留容器
docker builder prune -a      # 清掉半截缓存
docker compose build         # 重新构建
docker compose up -d
```

## 九、使用外部反向代理（1panel / OpenResty / Nginx）

如果服务器已有 Nginx/OpenResty 管理前端和 SSL：

### 9.1 暴露后端端口

`docker-compose.yml` 中取消注释：

```yaml
oa-backend:
  ports:
    - "127.0.0.1:8000:8000"
```

### 9.2 只启动后端

```bash
docker compose up oa-backend -d
```

### 9.3 前端静态文件

```bash
# 本地构建
cd frontend && npm run build

# 上传到服务器
scp -r frontend/dist user@server:/opt/oa-agent/frontend/dist/
```

### 9.4 外部 Nginx 反向代理示例

```nginx
server {
    listen 80;
    server_name oa.your-domain.com;

    # 前端
    root /opt/oa-agent/frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /health {
        proxy_pass http://127.0.0.1:8000;
    }
    location /generate {
        proxy_pass http://127.0.0.1:8000;
        proxy_buffering off;
        proxy_cache off;
    }
    location /oa/ {
        proxy_pass http://127.0.0.1:8000;
    }
    location /download/ {
        proxy_pass http://127.0.0.1:8000;
    }
}
```

## 十、常见问题

### Q1：前端页面空白 / 接口 502？

```bash
docker compose logs oa-backend   # 检查后端是否正常启动
docker compose logs oa-nginx     # 检查 Nginx 错误日志
```

### Q2：生成公文时看不到流式输出？

Nginx 反代 `/generate` 路径必须关闭缓冲。确认 `deploy/nginx.conf` 包含：

```nginx
proxy_buffering off;
proxy_cache off;
```

### Q3：报「知识库未构建」？

```bash
# 检查向量库文件是否存在
ls /opt/oa-agent/data/vector_db/

# 如为空，执行构建
docker exec -it oa-backend python scripts/run_api.py --build-kb
```

### Q4：Embedding 模型下载失败？

国内服务器可能无法访问 HuggingFace。确认 Dockerfile 使用 ModelScope 下载（已配置）：

```dockerfile
RUN python scripts/download_model.py \
    --model BAAI/bge-small-zh-v1.5 \
    --output /app/models/bge-small-zh-v1.5
```

### Q5：内存不足？

```bash
docker stats  # 查看实际使用量
```

在 `docker-compose.yml` 中限制资源：

```yaml
oa-backend:
  deploy:
    resources:
      limits:
        memory: 2G
```

### Q6：如何修改端口？

`docker-compose.yml` 中修改 Nginx 端口映射：

```yaml
oa-nginx:
  ports:
    - "8080:80"    # 改为 8080
```

### Q7：刷新页面出现 404？

确认 `deploy/nginx.conf` 中有 `try_files` 回退：

```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```
